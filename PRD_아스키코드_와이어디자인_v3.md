# CareerFolio 와이어프레임 (ASCII) — v3

- 기준 문서: PRD_커리어폴리오_v3.md
- 반영 사항: ① 또래(동일 연령대) 비교 분석 ② API 미구현 · 전 화면 목업(mock) 기반 ③ 1주 데모 일정 ④ 파일 저장소 = 사용자 Google Drive
- 표기: `[ ]` 버튼/토글, `[[ ]]` 주요 CTA, `----` 구분선, `<>` 아이콘 자리
- 화면 하단 `<demo>` 표시는 "실제 동작이 아닌 목업임을 알리는 안내 문구" 위치를 뜻함


## 4. 포트폴리오 빌더 (목업 초안) — Feature 3

```
+--------------------------------------------------------------------+
| < 목록    포트폴리오 초안 (마케팅 직무)     v1  [되돌리기] [[저장]]  |
+--------------------------------------------------------------------+
| 활동 순서    |  본문 (블록 에디터)                                   |
| ------------ | ---------------------------------------------------- |
| [::] 자기소개 |  ## 자기소개                                        |
| [::] 해커톤   |  데이터 기반 문제 해결에 강점이 있는 백엔드 개발자... |
| [::] 마케팅   |  [문단 선택 시 노출]                                 |
|      인턴     |   +--------------------------------------------+     |
| [::] 공모전   |   | AI 재작성: [더 간결하게] [성과 강조]         |     |
|               |   |            [직무 키워드 반영]                |     |
| [ ] 논문      |   +--------------------------------------------+     |
|  (제외됨)     |                                                      |
|               |  ## 교내 해커톤 (2024.03~2024.06)                    |
| + 활동 추가   |  API 응답 속도를 800ms에서 220ms로 72% 단축하며...   |
|               |  <경고 배지> 인터뷰 미완료 항목 있음 → [인터뷰 이동] |
|               |                                                      |
+--------------------------------------------------------------------+
|  [ PDF 내보내기 ]   [ 공유 링크 생성 ]        [[ 비교 분석 실행 ]]   |
|                                                                      |
|  <demo> 직무 선택 시 실제 생성 없이 미리 준비된 초안 템플릿이         |
|         채워집니다. 편집 UI는 실제 동작하나 저장되지는 않습니다      |
+--------------------------------------------------------------------+
```

## 5. 또래 비교 분석 (목업 리포트) — Feature 4

```
+--------------------------------------------------------------------+
| < 포트폴리오   또래 비교 분석                                        |
+--------------------------------------------------------------------+
|  비교 그룹 설정                                                      |
|  나이대: [ 만 24~26세 ▾ ]   직무: [ 마케팅 ▾ ]   표본 수: 128명      |
|  ------------------------------------------------------------------ |
|                                                                      |
|  종합 갭 스코어                    또래 그룹 대비 강점/약점 (레이더)  |
|  +------------------+             +----------------------------+    |
|  |                  |             |         정량성과            |    |
|  |      72 / 100    |             |        /        \           |    |
|  |   (또래 평균 68)  |             |   구성순서        문제해결력  |    |
|  |                  |             |        \        /           |    |
|  +------------------+             |         스토리텔링           |    |
|                                   |   -- 나  ·· 또래 평균         |    |
|                                   +----------------------------+    |
|                                                                      |
|  또래 그룹 공통 패턴                                                 |
|  ------------------------------------------------------------------ |
|  · 대부분 첫 문단에 정량 성과(수치)를 배치함                          |
|  · 활동 3~4개 중 1개는 "실패 후 개선" 스토리를 포함                   |
|  · 자기소개는 평균 3문장 이내로 간결하게 구성                         |
|                                                                      |
|  개선 제안 체크리스트                                                |
|  ------------------------------------------------------------------ |
|  [ ] 자기소개 첫 문장에 핵심 정량 성과 배치하기                       |
|  [ ] "마케팅 인턴" 항목에 실패/개선 사례 1건 추가하기                 |
|  [x] 활동 순서를 성과 크기순으로 재배열함                             |
|                                                                      |
|  <참고> 이 리포트는 익명화된 또래 그룹의 공통 패턴만 보여주며,        |
|         개별 사용자의 포트폴리오 원문은 노출되지 않습니다.            |
|                                                                      |
|                                    [ 리포트 PDF 저장 ]  [[ 초안 수정 반영 ]] |
|                                                                      |
|  <demo> 나이대·직무 조합별로 미리 정의된 값이 표시되는 목업           |
|         리포트입니다 (실제 또래 DB 검색 아님)                        |
+--------------------------------------------------------------------+
```
